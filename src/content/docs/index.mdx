## You know the drill

You want to build an AI agent. So you need an LLM. Which means you need inference infrastructure. And a vector database for memory. And an experiment tracker. And somewhere to store models. And CI/CD to deploy it. And monitoring to know when it breaks.

Three weeks later you're debugging Helm charts instead of writing code.

Or you pay per token, hit rate limits at the worst moment, and watch your data flow through someone else's servers.

**There's a third option.**

## One machine. Everything you need.

Thinkube turns a single GPU machine into a complete AI development platform. Not a collection of tools you wire together — an integrated environment where everything is pre-configured, authenticated, and talking to each other from the moment you install it.

## What a day looks like

**Morning.** Open JupyterLab in your browser. Tell Thinky: "Load yesterday's experiment from MLflow and try a different embedding model." It writes the code, runs it, compares the results, and shows you the graphs. You drink your coffee.

**Afternoon.** Switch to Code Server. Claude Code helps you build a FastAPI backend. Push to Git. The CI/CD pipeline builds your container, pushes to your private registry, and deploys it with a TLS certificate. Your app is live at `myapp.yourdomain.com` before you finish lunch.

**Evening.** Deploy a new Llama model through the dashboard. One click. It downloads, loads onto your GPU, and joins your LiteLLM gateway. Your agents can use it immediately.

No cloud bills. No cold starts. No permission requests. It's your machine.

## What's inside

| | |
|---|---|
| **AI Development** | JupyterHub + Thinky, Code Server + Claude Code, Thinkube Control dashboard |
| **LLM Inference** | Ollama, vLLM, TensorRT-LLM, LiteLLM gateway |
| **Vector Databases** | Qdrant, Weaviate, Chroma |
| **ML Platform** | MLflow experiment tracking + model registry, Langfuse LLM observability |
| **Infrastructure** | PostgreSQL, Valkey, ClickHouse, Harbor container registry |
| **Automation** | Gitea, Argo Workflows, ArgoCD, Prometheus + Perses |

Everything has SSO through Keycloak. Everything stores data on shared JuiceFS storage. Everything is monitored. You install once and it works.
