= Components
:page-type: reference

Thinkube ships a curated, integrated stack. Each component is a building block you
can reach from the control panel — or, mostly, from Claude in code-server.

[.cards]
* xref:reference.adoc[thinkube-control]
+
The web control panel and MCP server — deploy apps, manage models, GPUs, secrets.

* xref:concept.adoc[JupyterHub]
+
GPU notebooks for fine-tuning, with MLflow experiment tracking built in.

* xref:tutorial.adoc[code-server]
+
Browser VS Code — your cockpit, where you run Claude and drive the platform.

* xref:how-to.adoc[LLM Gateway]
+
Load models onto GPUs on demand, served behind an OpenAI-compatible API.

* xref:reference.adoc[Harbor]
+
The private image registry every build pushes to.

* xref:reference.adoc[Gitea + Argo]
+
Git, CI, and GitOps — push code, get a ~90s rebuild and redeploy.
