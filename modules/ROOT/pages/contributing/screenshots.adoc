== Where images land

Captured images are named `<id>.png` and committed into the owning component's
`modules/ROOT/images/` — the docs repo for §1/§3-docs/§4/§5/§6, the `control` repo
for `cp-overview`, and `thinkube-installer` for the installer set (docs-with-code).
When an image lands, swap the page's `[.screenshot-needed]` block for
`image::<id>.png[caption]`.

NOTE: The Playwright/Keycloak approach and the manual capture checklist live in
SP-tgiy46 SL-3 (added to this page).
