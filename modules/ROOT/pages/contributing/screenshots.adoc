== Capture plan (the follow-on spec)

Mocks are swapped for real assets by id. Two paths:

🤖 *Playwright set* — log in **once** through Keycloak (`auth.thinkube.com`) in the
Playwright MCP browser (you complete the login; the session carries), then for each
🤖 id navigate its URL and `browser_take_screenshot` to `<id>.png`. URLs come from
the control panel / `jupyter.thinkube.com` / `mlflow.thinkube.com` / the deployed
apps. Images replace `<id>.svg` with `<id>.png` in the owning component's
`modules/ROOT/images/`.

✋ *Manual checklist (yours)* — capture each from your logged-in session and drop the
file in by id:

* `cockpit-session` — code-server with a live Claude Code session running.
* `cockpit-mcp-call` — a Claude turn calling an MCP tool (e.g. `deploy_template`).
* `lab-notebook` — a JupyterHub notebook with the Thinky assistant panel open.
* `lab-finetune` — a fine-tuning notebook mid-run (loss/output visible).
* `cockpit-scaffold` — Claude scaffolding an app from `tkt-webapp-vue-fastapi`.
* `gpu-multiarch` — a terminal showing `nvidia-smi` + the node arch (arm64/amd64).
* *Installer walkthrough* — the **full current installer** (Tauri desktop app),
  one shot per screen, `installer-01…NN`, replacing the stale set in
  `thinkube-installer`.

== Where images land

Named `<id>.png`, committed into the owning component's `modules/ROOT/images/` — the
docs repo for most, the `control` repo for `cp-overview`, and `thinkube-installer`
for the installer set (docs-with-code).
