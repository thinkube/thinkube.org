== Scope boundary — what we do *not* document

Thinkube runs a lot of services — Gitea, Harbor, ArgoCD, MLflow, Keycloak,
JuiceFS, SeaweedFS, PostgreSQL, DevPI, Argo Workflows and the optional
components. **None of these get interface ("here's their dashboard") docs.**

* They appear only as **mechanism** in xref:how-it-works/components.adoc[§6 Platform components] —
  what they are and how the platform uses them — not as places you operate.
* Operationally you reach them through **Claude + MCP**, not their own UIs.
* The documented interface surface is the **three pillars + the loop**. That is
  the boundary; new service docs do not get added without revisiting this map.
